## 撰写指南

### 关键词优化
- 在标题中自然融入 1-2 个主要关键词
- 在首段和正文中合理分布相关关键词（不要堆砌）
- 在 `keywords` 字段中列出 5-8 个相关关键词，用逗号分隔

### 内容结构
- 使用清晰的标题层级（H2 -> H3 -> H4）
- 每个段落 3-5 句话，保持阅读流畅性
- 使用**粗体**强调关键概念
- 使用引用块突出重要信息

### SEO 最佳实践
- `description` 字段应该 150-160 字符，包含主要关键词
- 文章长度建议 800-2000 字
- 定期更新旧文章，保持内容新鲜度
- 在相关文章中相互链接，提高网站内部链接结构

### Front Matter 字段说明
| 字段 | 说明 | 示例 |
|------|------|------|
| `title` | 文章标题 | "Clash for Windows 完整教程" |
| `date` | 发布日期（格式：YYYY-MM-DD HH:MM:SS +08:00） | "2026-04-21 10:00:00 +08:00" |
| `permalink` | 文章 URL 路径 | "/clash-for-windows-guide/" |
| `categories` | 文章分类 | ["教程指南", "技巧"] |
| `tags` | 文章标签 | ["clash", "windows"] |
| `keywords` | SEO 关键词 | "clash,windows,教程" |
| `description` | 文章摘要（SEO 用） | "本文详细介绍..." |
| `author` | 文章作者 | "作者名称" |

### 命名规范
- 文件名格式：`YYYY-MM-DD-url-slug.md`
- URL slug 应使用英文或拼音，用连字符分隔
- 示例：`2026-04-21-clash-windows-guide.md`
