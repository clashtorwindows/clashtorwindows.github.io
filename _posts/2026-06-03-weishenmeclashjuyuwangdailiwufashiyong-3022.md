地址和端口来监听局域网内的连接请求。如果绑定错误（比如绑定了 127.0.0.1，也就是本地回环地址），其他设备自然无法连接。</li> <li><strong>代理模式或规则冲突</strong>：虽然可能性较低，但在规则模式（Rule）下，如果存在错误的规则意外阻止了局域网内的特定 IP 或连接，也可能导致问题。</li> </ul> <h4>2. 宿主设备的网络或防火墙问题</h4> <ul> <li><strong>系统防火墙阻止</strong>：运行 Clash 的电脑（宿主设备）的系统防火墙（如 Windows Defender、macOS 防火墙）可能阻止了来自局域网其他设备的连接请求到达 Clash 监听的端口。</li> <li><strong>网络接口绑定错误</strong>：Clash 可能绑定到了一个不用于局域网通信的网络接口上。</li> </ul> <h4>3. 局域网环境问题</h4> <ul> <li><strong>路由器设置</strong>：少数情况下，路由器自身的防火墙或隔离设置可能影响设备间的通信。</li> <li><strong>IP 地址冲突或不稳定</strong>：宿主设备的本地 IP 地址如果频繁变化，客户端设备指向的代理地址就可能失效。</li> </ul> <h4>4. 客户端设备的配置问题</h4> <ul> <li><strong>代理设置错误</strong>：尝试使用 Clash 代理的设备（客户端设备）需要正确配置其系统代理，指向宿主设备的本地 IP 地址和 Clash 监听的端口。任何一个字符不对都会导致连接失败。</li> <li><strong>客户端设备防火墙</strong>：虽然不常见，但客户端设备自身的防火墙也可能阻止其向宿主设备发起连接。</li> </ul> </ul> <p>修改设置后，通常需要重启 Clash 服务或者客户端才能生效。</p> <h4>第二步：检查宿主设备的系统防火墙</h4> <p>宿主设备的防火墙可能会阻止其他设备连接到 Clash 的端口。你需要允许 Clash 应用程序通过防火墙，或者手动添加入站规则，允许其他设备访问 Clash 监听的端口（通常是 HTTP 代理端口和 SOCKS 代理端口）。</p> <ul>
